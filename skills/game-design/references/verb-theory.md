# Verb Theory in Game Design

A comprehensive reference on the verb-based approach to game mechanics design, originating from Satoshi Tajiri's philosophy and expanded by multiple theorists.

---

## 1. Theory Overview

### Origin: Satoshi Tajiri's "Verbs as Themes"

Satoshi Tajiri, creator of Pokemon, articulated the foundational insight in his 1996 book "New Game Design" (Shin Game Design / Enix, 160 pages). He observed that Namco's early arcade games were each designed around a single specific action:

> "For example, with Dig Dug, the theme was to dig. I felt the same joy I had in junior high when I learned new English verbs. That inspired me to use verbs as themes."
> -- Satoshi Tajiri, "New Game Design" (1996)

This philosophy drove Tajiri to create his first game, Mendel Palace (1989), built around the single verb "flip" (flipping floor tiles). Even as scrolling games dominated, Tajiri insisted on designing from a core verb outward.

Similarly, Toru Iwatani designed Pac-Man (1980) around the verb "eat," deliberately choosing a single action as the entire game's foundation.

### The Core Insight

**Games are fundamentally collections of verbs.** A game's identity, feel, and depth emerge from which verbs the player can perform, how those verbs interact, and what situations they confront.

| Principle | Description |
|-----------|-------------|
| **Single-verb clarity** | The best games can be described by one or a few core verbs |
| **Verbs define genre** | Run + Jump = Platformer; Shoot + Cover = Shooter; Collect + Trade = RPG |
| **Verbs before objects** | Design the actions first, then design the objects that support them |
| **Verb depth** | A single verb applied to many varied situations creates deep gameplay |

## 2. Theoretical Framework

### 2.1 Satoshi Tajiri: Verbs as Themes (1996)

- Each game should be built around a core verb (single action)
- Classic arcade games exemplify this: Dig Dug = "dig," Pac-Man = "eat"
- The joy of discovering a new verb parallels the joy of learning a new word
- Source: "Shin Game Design" (Enix, 1996)

### 2.2 Anna Anthropy & Naomi Clark: A Game Design Vocabulary (2014)

- **Verbs** are rules that give the player freedom to change the game state
- **Objects** are the interactive elements verbs act upon
- Games are built on specific rules involving "verbs" (actions) and "objects" (interactions)
- Engaging choices arise when verbs are contextualized, pushing players to make tactical decisions
- The design of verbs affects both how a game feels and what it means
- Verbs are the most important of all game elements

### 2.3 Raph Koster: Game Grammar & Atoms (2005+)

- **Intent is expressed as a verb** in Koster's game grammar
- **Games nest**: the interactivity loop applies from a single button press to the overarching objective
- **Core loop**: verbs used in the loop should confront many varied situations; if a verb cannot, the core loop is flawed
- **Atoms**: the smallest building blocks (mechanics) that form learning loops
- **Napkin language**: currencies, verbs, and objects for game analysis
- Source: "A Grammar of Gameplay" and "Theory of Fun for Game Design"

### 2.4 Jesse Schell: The Art of Game Design (2008)

- Games are focused around a particular set of engaging verbs
- **Operational actions**: direct input (press button)
- **Resultant actions**: emergent outcomes from verb combinations
- **Elemental Tetrad**: Mechanics, Story, Aesthetics, Technology -- verbs live in Mechanics
- Verbs can be enumerated using the Lens of Actions: "What actions do players wish they could do in my game?"
- Source: "The Art of Game Design: A Book of Lenses"

## 3. Verb Classification Taxonomy

### 3.1 By Domain

| Category | Example Verbs | Game Examples |
|----------|--------------|---------------|
| **Movement** | Run, Jump, Climb, Swim, Fly, Dash, Dodge | Mario, Celeste, Spider-Man |
| **Combat** | Attack, Block, Parry, Shoot, Cast, Throw | Dark Souls, Doom, Street Fighter |
| **Collection** | Collect, Gather, Harvest, Loot, Catch | Pokemon, Animal Crossing, Diablo |
| **Social** | Trade, Talk, Ally, Betray, Negotiate | Pokemon, Diplomacy, Mass Effect |
| **Construction** | Build, Craft, Place, Combine, Upgrade | Minecraft, Factorio, Terraria |
| **Destruction** | Break, Destroy, Demolish, Dismantle | Teardown, Red Faction |
| **Exploration** | Explore, Discover, Search, Scan, Map | Zelda, Metroid, No Man's Sky |
| **Puzzle** | Solve, Rotate, Connect, Match, Arrange | Tetris, Portal, Baba Is You |
| **Management** | Manage, Assign, Prioritize, Schedule | Civilization, RollerCoaster Tycoon |
| **Stealth** | Sneak, Hide, Disguise, Distract | Hitman, Metal Gear Solid |

### 3.2 By Direction (Nathan Savant's Verb Directions)

| Direction | Description | Examples |
|-----------|-------------|----------|
| **Outward** | Acting upon external targets | Attack, Push, Throw |
| **Inward** | Responding to incoming actions | Block, Dodge, Absorb |
| **Self** | Modifying own state | Heal, Buff, Transform |
| **Environmental** | Acting on the world itself | Build, Destroy, Terraform |

### 3.3 By Priority (Implementation Triage)

| Priority | Label | Definition |
|----------|-------|------------|
| **Critical** | Must-have | Without this verb, there is no game |
| **Valuable** | Should-have | Loss requires major gameplay redesign |
| **Desirable** | Nice-to-have | Can be cut if time/budget runs out |

## 4. Concrete Examples

### 4.1 Classic Arcade Era (Single-Verb Games)

| Game | Core Verb | Designer |
|------|-----------|----------|
| Pac-Man | Eat | Toru Iwatani |
| Dig Dug | Dig | Namco |
| Donkey Kong | Jump (+ Run, Climb) | Shigeru Miyamoto |
| Space Invaders | Shoot | Tomohiro Nishikado |
| Breakout | Bounce | Nolan Bushnell / Steve Jobs |
| Frogger | Cross | Konami |

### 4.2 Multi-Verb Modern Games

**Super Mario Bros.**
- Core: **Run, Jump**
- Secondary: Stomp, Swim, Climb, Slide, Throw (Fireball)
- The jump verb alone confronts dozens of situations: gaps, enemies, blocks, heights

**The Legend of Zelda**
- Core: **Explore, Fight, Solve**
- Secondary: Slash, Shoot, Push, Pull, Bomb, Hookshot, Glide (BotW)
- Verb acquisition drives progression (Metroidvania pattern)

**Pokemon (Red/Green)**
- Core: **Collect, Battle, Trade**
- Secondary: Explore, Train, Evolve, Teach (moves), Breed (later gens)
- Tajiri designed the core around the collecting verb, inspired by insect collecting

**Zelda: Breath of the Wild**
- Core: **Explore, Climb, Cook, Fight**
- Verb combinations create emergent dynamics: Hit + Freeze + Climb = creative traversal

**Dark Souls**
- Core: **Attack, Dodge, Block**
- The inward verb (Dodge/Block) is equally important as the outward verb (Attack)

**Minecraft**
- Core: **Mine, Craft, Build, Explore**
- Verb-object interactions multiply possibilities exponentially

### 4.3 Board/Card Games

|------|-----------|
| Chess | Move, Capture |
| Checkers | Move Forward, Jump, Move Backward (kings) |
| Magic: The Gathering | Lay (Permanent), Tap, Attack, Play (Spell) |

## 5. Relationship to the MDA Framework

### MDA Overview

The Mechanics-Dynamics-Aesthetics (MDA) framework (Hunicke, LeBlanc, Zubek, 2004) formalizes game properties into three layers:

```
Designer perspective:  Mechanics --> Dynamics --> Aesthetics
Player perspective:    Aesthetics --> Dynamics --> Mechanics
```

### Where Verbs Fit in MDA

| MDA Layer | Relationship to Verbs |
|-----------|-----------------------|
| **Mechanics** | Verbs ARE the core mechanics. Each verb is a rule giving the player freedom to change game state |
| **Dynamics** | Verb combinations and verb-situation interactions produce emergent behavior at runtime |
| **Aesthetics** | The emotional response players feel from exercising verbs in context |

### Verb-MDA Design Flow

```
1. Define target Aesthetics (what should the player feel?)
   e.g., "Sense of exploration and wonder"

2. Identify Dynamics that produce those aesthetics
   e.g., "Discovering unknown areas, finding secrets"

3. Design Mechanics (verbs) that generate those dynamics
   e.g., Verbs: Explore, Climb, Glide, Scan
         Objects: Map, Terrain, Hidden Areas, Collectibles
```

### MDA Aesthetic Categories and Verb Alignment

| Aesthetic | Description | Supporting Verb Types |
|-----------|-------------|----------------------|
| Sensation | Sensory pleasure | Movement verbs with feel/feedback |
| Fantasy | Make-believe | Transformation, magic verbs |
| Narrative | Story as drama | Dialogue, choice verbs |
| Challenge | Obstacle course | Precision movement, combat verbs |
| Fellowship | Social framework | Trade, cooperate, communicate verbs |
| Discovery | Uncharted territory | Explore, search, unlock verbs |
| Expression | Self-discovery | Build, create, customize verbs |
| Submission | Pastime | Collect, grind, idle verbs |

## 6. Practical Workflow: Verb-Based Design

### Phase 1: Verb Extraction

Define the game experience by identifying its core verbs.

```
Step 1: Start with the target experience (feeling/fantasy)
        "I want the player to feel like a master thief"

Step 2: Brainstorm all relevant verbs (unconstrained)
        Sneak, Steal, Lockpick, Hack, Disguise, Climb, Crawl,
        Distract, Knock Out, Hide, Plan, Escape, Run, Observe,
        Bribe, Eavesdrop, Grapple, Swing, Photograph, Seduce

Step 3: Prioritize using Critical/Valuable/Desirable labels
        Critical:  Sneak, Steal, Hide
        Valuable:  Lockpick, Disguise, Distract, Climb, Observe
        Desirable: Hack, Bribe, Grapple, Photograph, Eavesdrop

Step 4: Select 2-4 core verbs
        Core verbs: Sneak, Steal, Disguise, Observe
```

### Phase 2: Verb-Object Matrix

Map verbs against objects to discover interactions.

```
           | Guard | Door | Camera | Safe | Costume | Shadow |
-----------|-------|------|--------|------|---------|--------|
Sneak      |  past |  --  |  past  |  --  |   --    |  into  |
Steal      |  keys |  --  |   --   | open |   --    |   --   |
Disguise   |  as   |  --  |  fool  |  --  |  wear   |   --   |
Observe    |  patrol| lock |  angle | combo|  --     |  hide  |
```

Each non-empty cell is a potential mechanic. Empty cells prompt design questions: "Should the player be able to steal from a camera?"

### Phase 3: Verb Combination Analysis

Explore how verbs interact with each other.

```
Observe + Sneak   = "Read patrol pattern, then time movement"
Disguise + Steal  = "Walk up openly, then pocket the item"
Observe + Steal   = "Find safe combo by watching NPC"
Sneak + Disguise  = "Switch costume in shadows for new access"
```

Verb combinations that create emergent dynamics are the highest-value design targets.

### Phase 4: Prototype Core Loop

Build a minimal prototype testing only the core verbs.

```
Prototype 1: Single room, one guard, one item
  - Can the player Sneak past the guard?
  - Can the player Steal the item?
  - Does it feel satisfying?

Prototype 2: Add Observe verb
  - Guard has a patrol pattern
  - Player can watch and time their approach
  - Does observation create meaningful decisions?

Prototype 3: Add Disguise verb
  - Player can change appearance
  - Creates alternative approach to same problem
  - Do the verbs create interesting choices?
```

### Phase 5: Verb Refinement

Iterate based on playtesting.

| Question | Action |
|----------|--------|
| Is a verb never used? | Cut it or make it more necessary |
| Is one verb dominant? | Nerf it or buff alternatives |
| Do situations feel samey? | Add more varied contexts for verbs |
| Is the game too simple? | Add verb combinations |
| Is the game too complex? | Reduce to fewer, deeper verbs |
| Does a verb feel bad? | Improve feedback/responsiveness |

### Phase 6: Verb Introduction Pacing

Plan how and when verbs are introduced to the player.

```
Tutorial Level: Sneak only
Level 2:       Sneak + Observe
Level 3:       Sneak + Observe + Steal
Level 4:       All core verbs + first combination challenge
Level 5+:      Increasingly complex combinations of all verbs
```

The Zelda series exemplifies this: each dungeon bestows a new verb (tool), and subsequent challenges require combining it with previous verbs.

## 7. Design Principles Summary

| # | Principle | Description |
|---|-----------|-------------|
| 1 | **Verbs before objects** | Design what the player does before designing what they do it to |
| 2 | **Fewer, deeper verbs** | A small set of verbs confronting many situations beats many shallow verbs |
| 3 | **Verbs define identity** | Your game IS its verbs. Mario IS "run and jump" |
| 4 | **Verb combinations = depth** | Emergent gameplay comes from verbs interacting with each other |
| 5 | **Every verb needs situations** | A verb that only applies to one scenario is wasted design space |
| 6 | **Verb feel matters** | The tactile feedback of executing a verb is as important as its function |
| 7 | **Introduce verbs gradually** | Teach one verb at a time, then combine (Zelda/Metroid pattern) |
| 8 | **Both directions matter** | Inward verbs (dodge, block) are as important as outward verbs (attack) |
